Developer instructions for qFeature

We assume that:

  1. You have forked the qFeature repository to another group or user
  2. You have cloned your fork to create a local copy
  3. You have configured your local copy to sync with the upstream at http://github.com/pnnl/qFeature:
    • git remote add upstream https://github.com/pnnl/qFeature
    • git remote -v

Note there are two active branches on qFeature: master and gh-pages. The master branch contains all the package, and the gh-pages branch contains the content that is viewable from http://pnnl.github.io/qFeature. It is imperative you not confuse these branches.

Summary of the editing process

Any time you plan to make edits on the master branch, please do the following:

  1. Make sure you are on the master branch of your local copy using:
    • git checkout master
  2. Sync your local copy with the upstream:
    • git fetch upstream
    • git merge upstream/master
  3. Make edits
  4. Process the package using prepPackage
  5. Commit your changes and push to the master branch of your fork on github
  6. On your fork on github, issue the pull request on the master branch
  7. After the pull request is merged to the upstream, you can sync your local copy again:
    • git fetch upstream
    • git merge upstream/master
  8. After syncing, there will be new commits that you'll want to push to your fork using
    • git push origin master
  9. The master branch of your local copy and your fork should now be "even" with the upstream at pnnl:master

Edits to the vignette

If, during your edits on the master branch, you make edits to the vignette, you will also need to update the vignette on the gh-pages branch, since the vignette is the index.html on the gh-pages branch. Please do the following:

  1. Start off on the master branch using:
    • git checkout master
  2. Copy the vignette file inst/doc/qFeatureTechnical.html to a temporary location
  3. Change the branch to the 'gh-pages' branch using:
    • git checkout gh-pages
  4. Sync your local copy with the upstream:
    • git fetch upstream
    • git merge upstream/gh-pages
  5. Move (and rename) the temporary copy of the vignette to index.html at the root of the gh-pages branch in your local copy
  6. Commit and push to the gh-pages branch of your fork
  7. On your fork on github, issue the pull request on the gh-pages branch
  8. After the pull request is merged to the upstream, you can sync your local copy again:
    • git fetch upstream
    • git merge upstream/gh-pages
  9. After syncing, there will be new commits that you'll want to push to your fork using
    • git push origin gh-pages
  10. The gh-pages branch of your local copy and your fork should now be "even" with the upstream at pnnl:gh-pages
